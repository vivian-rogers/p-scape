# Decisions (ADRs)

Tiny architecture decision records. One file per decision, numbered. Capture *why*, not *what* — the code shows what.

## Template

```markdown
# NNNN — Title

**Date:** YYYY-MM-DD
**Status:** proposed | accepted | superseded by NNNN

## Context
2–4 sentences on the situation that forced a choice.

## Decision
What we chose. One paragraph.

## Alternatives
Bullets. What else was on the table and why we passed.

## Consequences
What this commits us to. Any costs, follow-ups, or things to watch.
```

If a decision turns out wrong, don't delete the ADR — write a new one that supersedes it and update the old one's `Status`.
